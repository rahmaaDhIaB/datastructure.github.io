List
Think of it like a numbered playlist. Items are in order, each one has a number (index), and you can add, remove or change any song.
Cambridge calls this: ARRAY
What it looks like in memory

Always starts at index 0 — not 1. So a list of 5 items has indices 0, 1, 2, 3, 4.
The simplest way to think about it A list is like a row of boxes. Each box has a number on it starting from 0. You can put anything inside the boxes, look inside any box, change what's inside, or add more boxes at the end.
How to create a list

# List of strings

# Fixed size list — all slots set to -1 (Cambridge exam style)

# Full example
scores = [85, 72, 91, 60, 78]

print(scores[0])      # 85  — first item
print(scores[-1])     # 78  — last item
print(len(scores))    # 5   — how many items

scores.append(100)    # add 100 at the end
scores[1] = 99        # change index 1 from 72 to 99

for score in scores:
    print(score)        # prints each one
The #1 beginner mistake with lists A list of 5 items has indices 0, 1, 2, 3, 4. The last index is 4, NOT 5. Trying scores[5] on a 5-item list crashes with IndexError: list index out of range. Always remember: last index = len(list) - 1.

Dictionary
Think of it like a real dictionary or a contact list on your phone. You look something up by its name (key), not by a number.
Look up by name, not by number
The phone contacts analogy Imagine your phone contacts. To find Sara's number, you search for "Sara" — not "contact number 3". A dictionary works the same way. You give it a name (called a key) and it gives you back the value stored under that name. No index numbers needed.

# Read values
print(student["name"])   # Ali
print(student["age"])    # 17

student["score"] = 95

student["age"] = 18

if "score" in student:

for key in student:

List vs Dictionary — when to use which? Use a list when you have multiple items of the same type: a list of scores, a list of names.
Use a dictionary when you have one thing with multiple properties: one student with name, age, and grade.

Think of it this way: a list is a column. A dictionary is a row in a spreadsheet.

Tuple
Think of it like a list that's been sealed shut. Once you create it, you can read from it but you can never change it. Like GPS coordinates — once you've set them, they're locked.
Cambridge: no direct equivalent — use when data must NOT change
What a tuple looks like — round brackets, locked

🔒 Cannot be changed after creation
Why would you use a tuple instead of a list? Sometimes data should never be changed. For example: the days of the week, the months of the year, a GPS coordinate, RGB colour values. A tuple tells everyone: "this data is fixed — don't touch it." It also runs slightly faster than a list.
How to create a tuple
# A point in 2D space — should never change
point = (3, 7)

# RGB colour — red, green, blue (0-255)
red = (255, 0, 0)

# Days of the week — fixed, will never change
days = ("Mon", "Tue", "Wed", "Thu", "Fri")

# Unpacking — assign each value to its own variable
x, y = point      # x = 3, y = 7
r, g, b = red     # r = 255, g = 0, b = 0

A tuple is read-only. If you need to change data, use a list instead. 
# Tuple vs List — spot the difference
my_list  = [1, 2, 3]   # square brackets — changeable
my_tuple = (1, 2, 3)   # round brackets  — locked

my_list[0]  = 99       # ✓ works fine
my_tuple[0] = 99       # ✗ crashes — TypeError
Which one do I use?
One question — four answers. Pick the right structure for the job.
📋
Use a List when...
You have multiple items of the same type and order matters. A list of exam scores. A list of names. A queue. An array in Cambridge pseudocode.
scores = [85, 72, 91]
📖
Use a Dictionary when...
You have one thing with multiple properties. One student with name, age, grade. One product with price and stock. You look things up by name, not number.
student = {"name":"Ali"}
🔒
Use a Tuple when...
You have data that must never change. GPS coordinates. RGB colours. Days of the week. Months. You want Python to protect you from accidentally changing it.
point = (3, 7)
